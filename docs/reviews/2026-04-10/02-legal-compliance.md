# Legal and Regulatory Compliance Review — Bullpen
**Review date:** 2026-04-10  
**Reviewer role:** Legal/Compliance Analyst  
**Perspective:** Potential acquirer / independent assessment

---

## Executive Summary

Bullpen ships no LICENSE file, which is the most significant legal gap. The codebase has no terms of service, privacy policy, or export-control notices — appropriate for a private tool but required before any public distribution or commercial transaction. Data handling is minimal and local, reducing regulatory exposure. No obvious IP infringement was found.

## Detailed Findings

### L1 — HIGH: No LICENSE file

A search of the repository root and all subdirectories found no `LICENSE`, `LICENSE.txt`, `LICENSE.md`, or `COPYING` file. Without a license, the software is "all rights reserved" by default under copyright law in most jurisdictions. This means:

- No third party can legally use, copy, modify, or distribute the software.
- An acquirer cannot assert rights over the codebase without obtaining an explicit assignment.
- Contributors retain full copyright to their contributions with no grant to the project owner.

**Fix:** Add a LICENSE file. If this is intended for private use only, a proprietary license or a simple "Proprietary — All Rights Reserved" notice clarifies intent. For open-source distribution, choose an OSI-approved license (MIT, Apache 2.0, etc.).

---

### L2 — MEDIUM: No contributor license agreement

The repository has no CLA or Developer Certificate of Origin (DCO) process. If any contributions have come from outside the primary owner, the project owner may not hold sufficient rights to license or sell those contributions.

**Verification:** Git history shows commits attributed to multiple tools/agents (Claude Code). Anthropic's Claude Code usage terms govern outputs generated by Claude Code, and those terms should be reviewed to confirm the owner holds full rights to AI-assisted code before any acquisition or licensing transaction.

---

### L3 — LOW: CDN dependency licenses not documented

`static/index.html` loads five distinct libraries from CDN (Vue 3, Socket.IO, markdown-it, Prism, Lucide). Each carries its own license (Vue/Socket.IO/markdown-it: MIT; Prism: MIT; Lucide: ISC). These licenses are permissive and generally compatible with commercial use, but:

- No NOTICE or THIRD-PARTY-LICENSES file acknowledges them.
- An acquirer performing IP due diligence will expect an inventory.

**Fix:** Add a `THIRD-PARTY-LICENSES.md` listing each dependency, its version, and license.

---

### L4 — LOW: No export-control notice

`server/auth.py` uses Werkzeug's password hashing (scrypt/PBKDF2), which involves cryptographic algorithms. Shipping software with cryptographic functionality may require export classification under US EAR (Export Administration Regulations) even for open-source tools. For personal/internal use this is moot; for commercial distribution an EAR classification (typically ENC or ECCN 5D002) should be obtained.

---

### L5 — INFO: No privacy policy or terms of service

Bullpen stores no personal data beyond a username and password hash in `~/.bullpen/.env`. There is no analytics, telemetry, or network transmission of user data. For a private developer tool this gap is acceptable. If the tool were offered as a hosted service or bundled in a commercial product, a privacy policy and terms would be required.

---

### L6 — INFO: No NOTICE file

Apache 2.0 components (if any arise) require a NOTICE file. Current dependencies are MIT/ISC, which do not require one. This is informational only but good practice for any future compliance audit.

---

## Positive Controls

- No telemetry or analytics code found in the codebase.
- No third-party service integrations that would require data processing agreements.
- Data stays entirely local (`~/.bullpen/` and workspace `.bullpen/` directories).
- No user-identifying data beyond the optional username in the auth `.env` file.
- No trademark or brand assets of third parties embedded in the UI (Lucide icons are MIT/ISC licensed).
